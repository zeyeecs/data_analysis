# 单页定价仪表盘交互与可视化

## Parent

`.scratch/luxury-goods-analytics-dashboard/PRD.md`

## What to build

交付一个面向二奢收货定价场景的单页仪表盘界面，围绕“输入型号与成色后执行分析”这一主流程展开。页面应包含型号输入、成色选择、执行按钮、核心指标卡片、风险诊断提示以及 30 天价格趋势图，并正确处理加载态、空结果与错误态。

页面应直接消费上一切片提供的市场分析服务，不在客户端重复实现定价或风险算法。界面风格保持轻量、工作台式，不扩展为通用 admin 模板，也不增加与当前核心任务无关的导航结构。

本切片完成后，用户应可以在浏览器中完成一次完整分析并看见可解释结果。

## Acceptance criteria

- [ ] 页面支持输入型号编号并选择 `全新`、`95新`、`90新` 三档成色
- [ ] 用户触发分析后，可看到市场均价、建议最高收货价、风险等级三个核心结果
- [ ] 页面展示风险原因说明，并能区分正常结果与风险预警结果
- [ ] 页面展示最近 30 天价格趋势图，数据来自服务端返回的时间序列
- [ ] 页面具备清晰的加载态、无数据提示与错误提示
- [ ] 页面不直接访问数据库，所有业务数据都经由服务端分析接口返回
