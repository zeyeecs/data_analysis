# 快照累积保留，飞书目录不作唯一真相源

业务要求数据库长期保留多个 `snapshot_date` 的快照，以支持跨月的 **目录聚合对照** 与 **最近邻配对**；飞书 F/R/V 目录会定期清理，且清理往往不会在每次导入前完成。因此采用 **数据归档（累积保留）**：导入时追加新快照，同一 `snapshot_date` 再导入则 **快照覆盖** 该日数据；飞书上已删除、但曾导入过的文件对应数据仍留在库中。

**按日覆盖：** 同一表、同一 `snapshot_date` 再导入时，先删除该日全部行再写入；不以源文件名为并列维度保留多版本。

`import_to_tables.py` 默认**仅导入库中尚无的 `snapshot_date`**；`--reimport-all` 时对已有日期仍按 `snapshot_date` 删除后插入；仅 `--reset` 会重建空表。

**Considered Options:** 全量替换（每次导入只反映飞书目录当前文件）——与「飞书会删旧文件但仍需历史对照」冲突，故不采用。
