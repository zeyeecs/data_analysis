# 领域文档

工程类 skill 在探索代码库前应如何消费本仓库的领域文档。

## 探索前先读

- 仓库根目录的 **`CONTEXT.md`**，或
- 若存在 **`CONTEXT-MAP.md`**，按其中指针阅读各上下文的 `CONTEXT.md`
- **`docs/adr/`** — 阅读与当前改动相关的架构决策记录（ADR）

若上述文件尚不存在，**静默继续**，不要主动提醒创建；由 `/grill-with-docs` 在术语或决策落地时再补充。

## 目录结构

本仓库为**单上下文**：

```
/
├── CONTEXT.md          # 领域术语表（简体中文）
├── docs/adr/           # 架构决策（简体中文）
└── src/
```

## 使用术语表

在 issue 标题、重构建议、假设、测试名称中涉及领域概念时，使用 `CONTEXT.md` 中的定义，勿随意换用同义词。

术语表未覆盖的概念：要么改用已有术语，要么在 `/grill-with-docs` 中补充词条。

## ADR 冲突

若你的方案与已有 ADR 矛盾，应明确说明，例如：

> 与 ADR-0001 冲突——建议重新讨论，因为……

## 文档语言

新建或改写的 `CONTEXT.md`、`docs/adr/*.md` 及 `.scratch/` 内 Markdown **一律使用简体中文**（代码与路径除外）。
